API

以下是「建立 Google Cloud Platform 專案並啟用 Admin SDK API」的詳細操作步驟，適用於管理 Google Workspace 的自動化應用：

一、建立 Google Cloud Platform (GCP) 專案

1. 登入 Google Cloud Console
   前往 [Google Cloud Console](https://console.cloud.google.com)。

2. 建立新專案
   - 點選左上角的「專案選單」下拉按鈕（通常會顯示你目前的專案名稱）。
   - 在彈出的專案視窗中，點選「新建專案」。
   - 輸入專案名稱（如：workspace-account-automation）。
   - 選擇對應的組織與資料夾（如有需要）。
   - 點選「建立」。

3. 切換到新專案
   - 專案建立完成後，點選右上角的「通知」可快速切換到新專案，或再次於專案選單中切換至新專案。

二、啟用 Admin SDK API

1. 進入 API 管理頁面
   - 確認你當前專案已切換至剛建立的專案。
   - 在左側導覽列，選擇「API 與服務」→「程式庫」。

2. 搜尋並啟用 Admin SDK
   - 在搜尋框輸入 `Admin SDK`。
   - 點選搜尋結果中的「Admin SDK」。
   - 點選「啟用」。

三、（進階）建立並下載 Service Account 金鑰

1. 在左側選單選擇「IAM 與管理員」→「服務帳戶」。
2. 點選「建立服務帳戶」。
   - 填寫服務帳戶名稱與描述。
   - 點選「建立並繼續」。
3. 指定「角色」：通常選擇「專案」→「編輯者」或根據最小權限原則選擇適合的角色。
   - 點選「繼續」→「完成」。
4. 於服務帳戶列表找到剛才建立的帳戶，點選「動作」圖示 →「管理金鑰」。
5. 點選「新增金鑰」→ 選擇「JSON」→「建立」。
   - 會自動下載金鑰檔案（如 `service-account.json`）。

四、（必要）啟用網域代理授權（Domain-wide Delegation）

1. 在服務帳戶管理頁面，點選該帳戶名稱，進入詳細設定。
2. 編輯並勾選「啟用 G Suite 網域範圍授權」。
3. 記錄下「唯一識別碼（Client ID）」。

五、（必要）在 Google Workspace 管理控制台授權 API Scopes

1. 以管理員身份登入 [admin.google.com](https://admin.google.com)。
2. 前往「安全性」→「API 控制權」→「管理第三方應用程式存取」。
3. 點選「新增」→ 輸入剛才記錄的 Client ID。
4. 指定授權範圍（Scopes），如：
   https://www.googleapis.com/auth/admin.directory.user 
5. 儲存設定。

補充說明

- Admin SDK API 官方文件：[Admin SDK API documentation](https://console.developers.google.com/apis/library/admin.googleapis.com )
- 操作過程中如遇權限不足，請確認 Google Workspace 及 GCP 帳號均具備管理員權限。

如需更進階設定（如 OAuth Client ID 建立、權限最小化等），可進一步詢問。
