Git_ex2

三人協同開發 Reeborg 程式：Git/GitHub 詳細流程與指令

專案開發分工

組員 負責內容 Git 分支名稱
Alice 基本移動與轉向（new_move、turn、turn_right） feature/move-turn
Bob 收成邏輯（harvest_one_row、object_taken） feature/harvest
Carol 方向判斷與主流程（is_facing_east、主執行流程） feature/main-loop

Step 1：建立 GitHub 專案並初始化

  1. 任務：由一位組員（如 Alice）在 GitHub 新建 repo（如 reeborg-harvest）。
  2. 本地初始化： 
    git clone https://github.com/your-org/reeborg-harvest.git
cd reeborg-harvest
  3. 建立主分支（main）： 
    git checkout -b main
git push -u origin main
  4. 三位組員各自分出功能分支： 
    git checkout -b feature/move-turn      # Alice
git checkout -b feature/harvest        # Bob
git checkout -b feature/main-loop      # Carol
  5. 推送分支到遠端： 
    git push -u origin feature/move-turn      # Alice
git push -u origin feature/harvest        # Bob
git push -u origin feature/main-loop      # Carol

Step 2：組員各自開發（分支作業）

  1. 組員在各自分支中開發： 
    # 編輯程式檔案（如 reeborg.py）
# 開發完成後，確保功能可運作

git add reeborg.py
git commit -m "實作 new_move, turn, turn_right 函式"      # Alice
git commit -m "實作 harvest_one_row 與 object_taken 邏輯"  # Bob
git commit -m "實作 is_facing_east 與主流程"               # Carol
  2. 將更動推上 GitHub： 
    git push                                 # 各自分支

Step 3：建立 Pull Request（PR）並 Code Review

  1. 組員在 GitHub 建立 PR，從 feature 分支合併至 main：
    • 標題範例：「[feature/move-turn] 新增移動與轉向函式」
    • 描述本次更動內容，標註需審查組員。
  2. 其他組員 Review 程式碼：
    • 檢查有無語法錯誤、重複邏輯、註解清楚。
    • 提出建議，必要時請作者修正再 push。
  3. 審查通過後合併 PR：
    • 管理員或 PR 作者可點選「Merge pull request」。
  4. main 分支同步至本地： 
    git checkout main
git pull

Step 4：整合程式範例

def turn(n):
    for i in range(n):
        turn_left()
        
def turn_right():
    for i in range(3):
        turn_left()
        
def new_move(n):
    for i in range(n):
        move()
        
def harvest_one_row():
    global object_taken
    while object_here() and object_taken < 36:
        take()
        object_taken += 1
    # 只要沒達到 36 就繼續 move
    if object_taken < 36:
        move()
        
def is_facing_east():
    # 記錄原本面向的方向
    count = 0
    while not is_facing_north():
        turn_left()
        count += 1
        if count == 4:
            break
    # 轉到北方後，左轉三次就會面向東
    turn(3)
    return True

object_taken = 0
# move to the field
new_move(2)
turn_left()
new_move(2)

done = False
while is_facing_east() and not done:
    for i in range(2):
        # 面向東直走收成
        for j in range(6):
            harvest_one_row()
            if object_taken >= 36:
                done = True
                break
        if done:
            break
        for k in range(2):
            turn_left()
            move()
        # 面向西直走收成
        for j in range(6):
            harvest_one_row()
            if object_taken >= 36:
                done = True
                break
        if done:
            break
        for k in range(2):
            turn_right()
            move()
    if object_taken >= 36:
        print("task completed!")
        break

Step 5：後續協作建議

  • 每次開發新功能或修 bug，請先拉新分支，避免直接在 main 分支作業。
  • 合併前先 git pull origin main，解決衝突後再 push。
  • 務必在 PR 中詳細描述修改內容與測試方式，方便審查。
  • 善用 git log、git status、git diff 檢查歷史與更動。

常用 Git 指令說明

  • git checkout -b 分支名：從目前分支新建並切換到分支
  • git add 檔名：加入暫存區，準備提交
  • git commit -m "訊息"：提交更動，寫下本次修改說明
  • git push -u origin 分支名：推送分支到遠端
  • git pull：拉下遠端最新更動並合併
  • git merge 分支名：將指定分支合併到目前分支
  • git status：顯示目前狀態、尚未提交的變更
  • git log：查詢提交歷史
  • git diff：顯示檔案內容變更差異

結論

本範例展現三人協同開發的分工、分支管理、PR 審查流程，以及 Git/GitHub 實用指令。遵循此流程能大幅提升團隊協作及程式碼品質。
